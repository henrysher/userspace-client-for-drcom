Reporting Bugs

You have found a bug? Here are 5 steps to getting it fixed!

  1. Check your version

    To make sure you're not wasting your (and our) time, you should be using the latest version of drcom-client before you file your bug. First of all, you should download the latest build to be sure you have the latest version. If you've done this and you still experience the bug, go ahead to the next step.

  2. Search Issue List

    Now that you have the latest drcom-client version and still think you've found a drcom-client bug, search through Tickets first to see if anyone else has already filed it. This step is very important! If you find that someone has filed your bug already, please go to the next step anyway, but instead of filing a new bug, comment on the one you've found. If you can't find your bug in Issue List, go to the next step.

  3. File the bug!

    Now you are ready to file a bug on the drcom-client. The Writing a Good Bug Report document gives some tips about the most useful information to include in bug reports. The better your bug report, the higher the chance that your bug will be addressed (and possibly fixed) quickly!

  4. What happens next?

    Once your bug is filed, you will receive email when it is updated at each stage in the bug life cycle. After the bug is considered fixed, you may be asked to download the latest version and confirm that the fix works for you.
