2009-06-03: drcom-client.ORG 上线!

感谢一直支持我们的朋友，你们的帮助与激励，使drcom-client.org 终于上线了!

2009-05-25: drcom-client PUM v1.0 发布

支持 GNU/Linux 系统

新功能:
  • 简单易用的图形界面，方便用户配置
  • 支持现有的Dr.COM协议，(除v3.72_u31_2227之外)
  • 内网地址处理自动化
  • 消息提示，提供多语言支持(中文/英文)
  • 用户态协议，用Python实现，缩减了开发的时间

2009-03-28: drcom-client v1.4.8.2 发布

支持 GNU/Linux 系统

Bug修正以及部分修改:
  • 增加了英文版的README
  • 修正了内网地址的获取过程

2008-11-02: drcom-client v1.4.8.1 发布

支持 GNU/Linux 系统

Bug修正以及部分修改:
  • 修正认证模块中针对FTP等协议序列号调整的问题

2008-10-22: drcom-client v1.4.8 发布

支持 GNU/Linux 系统

Bug修正以及部分修改:
  • 已支持 Linux 2.6.27 内核
  • 增加readconf.c中缓存区空间

2008-05-17: drcom-client v1.4.6 发布

支持 GNU/Linux 系统

Bug修正以及部分修改:
  • 将 strtol() 改为 atoi()
  • 修正了recv fail的问题
  • 已支持 Linux 2.6.25 内核
  • 修正: 登录过程服务器回复包的长度可能小于(struct drcom_acknowledgement)的长度

2008-04-12: drcom-client v1.4.4 发布

支持 GNU/Linux 系统

Bug修正以及部分修改:
  • 增加了 README 文件
  • 修正: 配置文件中的MAC地址可能会使系统MAC地址失效

2008-04-05: drcom-client v1.4.2 发布

支持 GNU/Linux 系统

Bug修正以及部分修改:
  • 修正了UDP认证前缀的异常情况
  • 登录回应中读取内网地址

2008-03-31: drcom-client v1.4.0 发布

支持 GNU/Linux 系统

新功能:
  • 增加了基于netfilter的内核模块，解决了TCP/UDP认证前缀问题
  • 在drcom-client v1.3.7的基础上重写了整个实现结构
Bug修正以及部分修改:
  • 采用pthread_cancel()函数, 以避免内存泄露
  • 将老的 md5.c/md5.h 文件更新
